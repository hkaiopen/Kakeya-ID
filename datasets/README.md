# Kakeya Sets in 3D, 4D, and 5D – A Brief Introduction

## What is a Kakeya set?

Imagine you have a needle and you want to rotate it in every possible direction. How much space does the needle need to sweep out? Surprisingly, the answer can be "almost none." A **Kakeya set** is a region of space that contains a unit line segment pointing in every direction, yet its total volume can be made arbitrarily small.

This counter‑intuitive fact was discovered a century ago and has since become one of the deepest problems in mathematics, connecting geometry, harmonic analysis, and even theoretical physics.

## What is in this dataset?

This folder contains **the world's first publicly available numerical Kakeya-type sets in four and five dimensions**, along with a three‑dimensional reference.

| File | Dimension | Grid Size | Support Points | Measure Fraction |
| :--- | :--- | :--- | :--- | :--- |
| `kakeya_3d_N32_puregl.npz` | 3D | 32×32×32 | 13 | ~0.04% |
| `kakeya_4d_N24_puregl.npz` | 4D | 24×24×24×24 | 9 | ~0.0027% |
| `kakeya_5d_N16_puregl.npz` | 5D | 16×16×16×16×16 | **1** | ~0.0001% |

Each `.npz` file contains:
- `grid_shape`: the size of the computational grid.
- `dx`: the physical spacing between grid points.
- `L`: the side length of the periodic box.
- `support`: a Boolean mask indicating which grid points belong to the Kakeya set.
- `intensity`: the field amplitude at every grid point (for visualization or further analysis).

## Why are these datasets important?

- **First of their kind**: No publicly available dataset of a 4D or 5D Kakeya set existed before. These files provide concrete, verifiable examples of a famously elusive mathematical object.
- **Ultimate compression in 5D**: The five‑dimensional set consists of **a single grid point**, yet every randomly tested direction still intersects it (thanks to periodic boundaries and dense orbits). This is the most extreme compression possible on a discrete grid.
- **Verifiable**: The repository includes a script (`verify_kakeya.py`) that anyone can run to check the two defining properties:
  1. The total volume is extremely small.
  2. Randomly chosen directions all contain a unit segment that hits the set.

## How were they created?

The sets were generated by simulating a **generalized Ginzburg–Landau equation**—a type of partial differential equation originally used to model superconductors and phase transitions. The equation naturally drives the system toward a state of minimal volume while preserving directional information. The code used to produce the datasets is available in the `code/` folder.
